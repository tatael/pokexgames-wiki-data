Bundle estático da wiki

pokexgames-wiki-data

Referência publicada para quem consome o bundle da wiki do PokeXGames. Use esta página para inspecionar o manifesto, navegar pelos JSONs gerados e confirmar os caminhos de transporte expostos aos consumidores.

{} manifest.json Índice principal do bundle com versão de schema, categorias, resumos de páginas e o pagePath canônico de cada item. [] discovered-pages.json Artefato de revisão/debug que mostra as páginas descobertas recursivamente a partir das páginas-semente configuradas. :: Repositório Scraper, pipeline de normalização, inventário de origem e regras de validação usadas para publicar este bundle.

Como funciona

  1. O GitHub Actions executa um fluxo diário de sincronização e publicação, regenerando o manifest.json e os arquivos aninhados em pages/....
  2. Consumidores devem buscar primeiro o manifest.json e depois seguir o pagePath de cada entrada para carregar o JSON real da página.
  3. Não tente adivinhar caminhos a partir do slug. O manifesto é a fonte de verdade dos caminhos de transporte.
  4. Se uma atualização ou publicação falhar, os apps consumidores podem continuar usando sua cópia local em cache até que uma sincronização posterior seja concluída com sucesso.

Notas

Transporte vs cache local

Este bundle não inclui caminhos específicos de cache do app. Os consumidores devem sintetizar seus próprios campos locais e persistir um cache de fallback.

Idiomas

Os idiomas publicados são pt-BR, en e es. Por enquanto, os textos fora do português espelham o conteúdo normalizado em português.

Estrutura de caminhos

Os arquivos são publicados de forma aninhada por categoria e hierarquia, por exemplo pages/professions/estilista/...json.
